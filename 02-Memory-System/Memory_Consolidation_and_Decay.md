# Memory Consolidation and Decay v1.0

## Purpose

This document defines how North Vector strengthens, compresses, archives, and deactivates memories over time.

The objective is to prevent memory bloat while preserving the information most useful for future judgment.

## Core Principle

North Vector should become wiser over time, not merely larger.

Raw experiences should gradually become patterns, lessons, and principles.

## Consolidation Overview

Raw Events
↓
Repeated Observations
↓
Pattern Candidate
↓
Validated Pattern
↓
Lesson
↓
Principle

Consolidation transforms many low-level records into fewer high-value memories.

## What Should Be Consolidated

### Behavioral Patterns

Examples:
- repeated procrastination triggers
- successful study methods
- recurring decision mistakes

### Project Lessons

Examples:
- bottlenecks
- failed approaches
- successful execution methods

### Academic Lessons

Examples:
- effective review strategies
- workload estimation patterns
- performance trends

### Relationship Insights

Examples:
- communication preferences
- follow-up patterns
- trust changes

### Strategic Lessons

Examples:
- which opportunities create leverage
- which commitments reduce optionality

## Consolidation Rules

A consolidated memory should:
- summarize repeated evidence
- preserve links to source memories
- state confidence
- remain revisable
- avoid overstating one-time observations

One event should not automatically become a durable pattern.

## Consolidation Cadence

### Daily

Capture observations and minor lessons.

### Weekly

Review repeated events and possible patterns.

### Monthly

Consolidate project, academic, relationship, and behavioral insights.

### Semesterly

Create major academic and strategic summaries.

### Annually

Review identity, long-term goals, major life changes, and durable principles.

## Memory Reinforcement

A memory strengthens when:
- it is supported by new evidence
- Nishad confirms it
- it successfully predicts outcomes
- it repeatedly improves recommendations

Reinforcement may increase:
- confidence
- importance
- retrieval weight
- review interval

## Memory Decay

Decay reduces the retrieval priority of memories that are no longer useful.

Decay should not immediately delete information.

It should first reduce activation.

## Decay Categories

### No Decay

Examples:
- Constitution
- core values
- life mission
- user-authorized permanent records

### Slow Decay

Examples:
- preferences
- established behavioral patterns
- important relationship context

### Moderate Decay

Examples:
- active projects
- semester-specific plans
- temporary strategic goals

### Rapid Decay

Examples:
- completed errands
- temporary reminders
- minor short-term observations

## Decay Factors

Memories should lose active weight when:
- they have not been reinforced
- they are no longer relevant
- they are contradicted by newer evidence
- their deadline or usefulness has passed
- a newer consolidated memory replaces them

## Recency Is Not Everything

Older memories should not decay solely because they are old.

Importance and predictive value matter more than age.

A ten-year-old core value may remain more important than a recent temporary preference.

## Compression

Large collections of raw records should be compressed into summaries.

Example:

Forty study session records
↓
Semester study summary
↓
Long-term learning pattern

Compression should preserve:
- conclusions
- confidence
- notable exceptions
- source links

## Archive Policy

Memories should be archived when they remain historically useful but no longer belong in normal reasoning.

Examples:
- completed courses
- old jobs
- expired opportunities
- finished projects
- past travel plans

Archived memories remain searchable.

They should not enter ordinary context unless requested or historically relevant.

## Deletion Policy

Deletion should be rare.

Valid reasons include:
- user request
- privacy requirement
- duplicate record
- incorrect record with no historical value
- meaningless temporary data

Archiving is preferred when history may still matter.

## Contradiction and Decay

When new evidence conflicts with an existing memory:

1. Mark the conflict.
2. Lower confidence if appropriate.
3. Preserve historical versions.
4. Seek confirmation when the issue is important.
5. Replace the active memory only when evidence supports revision.

## Consolidated Memory Example

Source observations:
- Delayed English paper because topic was unclear.
- Delayed research application because requirements were vague.
- Started chemistry review quickly after receiving a concrete checklist.

Consolidated memory:
`Nishad is more likely to begin difficult work when the first action is concrete and clearly defined.`

Recommended response:
`Convert vague work into a visible first step.`

↓

↓
Principles

Example:

Event:
Missed preparation target.

Pattern:
Workload was underestimated.

## Phase 1 Implementation

Phase 1 should use simple rules:
- review candidate behavioral memories weekly
- archive completed operational memories
- reduce retrieval weight after expiration
- consolidate repeated observations manually or semi-automatically
- preserve source links
- never delete important memory automatically

## Success Criteria

The system succeeds if:
- memory volume remains manageable
- high-value lessons become easier to retrieve
- old operational clutter leaves active memory
- important history remains searchable
- outdated beliefs are revised
- predictive value improves over time

## Final Principle

A poor memory system stores more and more.

A strong memory system extracts meaning.

North Vector should accumulate understanding, not clutter.
