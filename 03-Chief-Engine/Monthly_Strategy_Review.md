# Monthly Strategy Review v1.0

## Purpose

This document defines how North Vector conducts a monthly review of strategy, priorities, resource allocation, goal alignment, and system performance.

The Monthly Strategy Review sits above the Weekly Review.

Weekly reviews ask whether execution is working.

Monthly reviews ask whether the strategy itself still makes sense.

## Core Principle

Execution can be disciplined and still point in the wrong direction.

North Vector must regularly review not only whether Nishad is doing things well, but whether he is doing the right things at all.

## Review Frequency

Default:
- once per month

Suggested timing:
- last weekend of the month
- first day of the new month
- another consistent monthly review block

## Inputs

The review should use:
- Constitution
- Mission Brief
- active goals
- weekly review summaries
- current risks
- current opportunities
- project status
- financial status
- academic performance
- health patterns
- relationship health
- behavioral trends
- system performance metrics

## Monthly Review Structure

### 1. Executive Strategic Summary

Summarize the month in a few sentences.

Example:
`Academics remained on track, research outreach improved, and North Vector documentation expanded. Project execution lagged behind planning, and sleep quality declined during the final week.`

### 2. Mission Alignment

Ask:
- Is the current Mission Brief still accurate?
- Are current priorities appropriate for this season of life?
- Has the surrounding context changed?

### 3. Goal Portfolio Review

Review all active strategic and operational goals.

For each goal, assess:
- alignment
- progress
- current priority
- resource cost
- risk status
- opportunity value
- whether it should continue, change, pause, or end

### 4. Resource Allocation Review

Evaluate where time, money, energy, and attention went.

Questions:
- Did resource use match declared priorities?
- Which domain received too much?
- Which domain was neglected?
- What generated the highest return?

### 5. Academic Strategy Review

Evaluate:
- course performance
- study systems
- workload estimates
- preparation timing
- academic risks

The goal is to improve strategy before problems compound.

### 6. Career Strategy Review

Evaluate:
- clinical experience
- research progress
- mentorship
- shadowing
- networking
- application readiness

### 7. Financial Strategy Review

Evaluate:
- income
- spending
- savings
- debt
- credit development
- investment behavior

### 8. Health Strategy Review

Evaluate:
- sleep
- exercise
- recovery
- stress
- burnout risk

Health is a strategic asset, not a maintenance afterthought.

### 9. Relationship Strategy Review

Evaluate:
- family maintenance
- friendships
- mentorship relationships
- professional relationships
- neglected follow-ups

### 10. Project Portfolio Review

Review all active projects.

Questions:
- Which projects are moving?
- Which are stalled?
- Which should be paused?
- Which are distractions?
- Which deserve more attention?

This section should actively guard against project sprawl.

### 11. Risk Landscape Review

Review:
- recurring risks
- resolved risks
- compound risks
- risks that were repeatedly underestimated

### 12. Opportunity Landscape Review

Review:
- captured opportunities
- missed opportunities
- emerging opportunities
- opportunities that should be pursued next month

### 13. Behavioral Trend Review

Evaluate:
- recurring failure modes
- recurring success patterns
- changes in behavior
- evidence that old assumptions are no longer accurate

### 14. System Performance Review

Evaluate North Vector itself.

Questions:
- Which recommendations were useful?
- Which warnings were too weak or too strong?
- Which memories were inaccurate?
- Which parts of the system created friction?
- What should be improved?

### 15. Strategic Decisions

Produce explicit decisions for the next month.

Examples:
- pause a low-value project
- increase chemistry study buffer
- prioritize research outreach
- reduce work hours
- protect sleep more aggressively

### 16. Next Month Priorities

Select:
- one primary strategic priority
- up to three major supporting priorities
- one domain to protect
- one behavior to change

## Portfolio Rules

North Vector should challenge overcrowding.

If too many strategic goals are active at once, Chief should recommend:
- deferring
- sequencing
- combining
- abandoning

Focus is a strategic resource.

## Drift Detection

The review should detect:
- actions no longer aligned with stated goals
- projects consuming disproportionate attention
- maintenance of outdated plans
- recurring tradeoffs that violate the Constitution

## Escalation Rule

If the same strategic problem appears across three monthly reviews:
- create a formal intervention plan
- consider changing the system, environment, or goal itself
- avoid treating the issue as a temporary lapse

## Example Monthly Strategic Decision

Observation:
North Vector documentation expanded substantially, but implementation did not begin.

Interpretation:
Architecture without execution is active.

Decision:
Freeze new architecture documents for two weeks and require a working prototype milestone.

## Failure Modes

### Strategy Theater

The review produces impressive language but no decisions.

### Goal Hoarding

Too many goals remain active.

### Metric Obsession

Numbers replace judgment.

### Short-Term Bias

Monthly urgency displaces long-term direction.

### Inertia

Misaligned goals continue because they already exist.

### System Blindness

North Vector evaluates Nishad but does not evaluate itself.

## Phase 1 Implementation

Phase 1 should support:
- monthly review generation
- goal portfolio review
- project portfolio review
- resource allocation review
- strategic decisions
- next-month priorities
- system performance review
- links to weekly reviews and long-term goals

Advanced trend analysis can come later.

## Success Criteria

The Monthly Strategy Review succeeds if Chief can reliably answer:
- Is the current strategy working?
- Are priorities still correct?
- What should receive more attention?
- What should receive less?
- Which goals should change?
- What should be stopped?
- Is North Vector itself becoming more useful?

## Final Principle

Weekly reviews improve execution.

Monthly reviews improve direction.

North Vector needs both.
